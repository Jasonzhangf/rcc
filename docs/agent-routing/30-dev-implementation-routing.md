# 开发闭环路由

## 索引概要
- L1-L6 `scope`：何时进入本路由。
- L8-L17 `gate`：进入实现前必须满足的条件。
- L19-L29 `sequence`：开发顺序。
- L31-L34 `anti-patterns`：禁止做法。

## 覆盖范围
适用于：准备进入任何实现动作之前，确认本阶段是否已具备闭环前置条件。

## 进入实现前的门禁
1. 已明确任务目标、范围、成功标准。
2. 对应 docs 真源已落盘或已更新。
3. 对应 skill 已存在或已补齐。
4. 已定义最小验证方式与完成证据。
5. 已在任务系统中记录当前工作项。

## 开发顺序
1. 更新 docs 真源，锁定边界与流程。
2. 更新或新增 skill，把可复用动作沉淀下来。
3. 只做当前阶段要求的最小实现。
4. 运行验证脚本/测试，显式暴露缺口。
5. 用证据关闭子任务，再决定是否进入下一阶段。

## 反模式
- 跳过 docs 直接写代码。
- 写了规则却没有 skill/验证入口。
- 用人工口头说明替代测试证据。
- 一个阶段未闭环就提前扩到下一阶段。
